Dataset and code availability

G. Raynaud

Open science is a key topic for today's research, as recent examples have depicted the difficulty of results reproductibility [1] in several scientific fields.  More specifically, the long-term disponibility of experimental data is discussed in an interesting article [2] that leads to some proposals, for instance : 

  • A better identification of researchers, especially to be able to contact them even when the email address from the academic institution does not exist anymore
  • Citable publications for dataset so that one does not need to carry out experiments or simuations that were already achieved. Also, it helps the acknowledgement of data generation work. It could also encourage to prepare, store and document data in a more structured and long-term strategy.

As an example, I looked for a while for simulations data regarding a very basic configuration (2D laminar incompressible flow over a cylinder in a channel, a basic benchmark). I hoped to find geometry files, mesh and even, with a bit of confidence, results of computations with famous solvers (for instance Ansys, Abaqus, openFOAM...), from which I could start quickier my work. But in the end, I had very few success for this simple CFD case. This is annoying when one knows how much it can be time and power consuming to validate CFD results.

Hence you will find on my github repository [3] some reports alongside numerical codes and data of my previous projects, when this is possible. When not specified otherwise, content is under licence GPL v3.0 [4] (among other things : free to use, modify and share, no warranty provided, attribution to authors).
